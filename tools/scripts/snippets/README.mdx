# Scripts

Scripts are located in `tools/scripts/snippets/`. These are command-line tools for generating and fetching documentation assets.

<Info>
All scripts use **git-based repo root detection** with fallback to `paths.config.json`. They can be run from any directory within the repository.
</Info>

---

## Configuration

### paths.config.json

Central configuration file containing all path definitions used by scripts.

**Purpose:** Fallback when `git rev-parse --show-toplevel` fails (e.g., when not in a git repo).

**Structure:**
```json
{
  "paths": {
    "snippets": "snippets",
    "snippetsExternal": "snippets/external",
    "snippetsGenerated": "snippets/generated",
    "snippetsComponents": "snippets/components",
    "snippetsWikiComponentLibrary": "snippets/snippetsWiki/componentLibrary/index.mdx",
    "docsJson": "docs.json",
    "v2Pages": "v2/pages",
    "lptExchanges": "v2/pages/06_delegators/token-resources/lpt-exchanges.mdx",
    "aiWorkerApi": "ai/worker/api"
  }
}
```

## API & Content Generation

### generate-api-docs.sh

Generates Mintlify API documentation from an OpenAPI specification file.

**What it does:**
1. Reads an OpenAPI spec (YAML or JSON)
2. Creates a **landing page** with CardGroups linking to each endpoint (grouped by tags)
3. Creates **individual MDX pages** for each endpoint with `openapi: METHOD /path` frontmatter
4. Outputs a **docs.json navigation snippet** ready to copy-paste

**Usage:**
```bash
./tools/scripts/snippets/generate-api-docs.sh <openapi-spec> <output-dir> <api-name> [github-repo-url]
```

**Example:**
```bash
./tools/scripts/snippets/generate-api-docs.sh ai/worker/api/openapi.yaml v2/pages/04_gateways/api-reference/AI-API "AI API" "https://github.com/livepeer/ai-worker"
```

**Output structure:**
```
output-dir/
├── ai-api.mdx           # Landing page with Base URLs + CardGroups
├── text-to-image.mdx    # openapi: post /text-to-image
├── image-to-image.mdx   # openapi: post /image-to-image
└── ...
```

**After running:** Copy the outputted JSON snippet into your `docs.json` navigation.

### generate-seo.js

Automatically generates and updates SEO metadata for MDX documentation pages.

**What it does:**
1. Scans authored MDX content in `v2/`, `docs/`, `docs-guide/`, `contribute/`, and `snippets/pages/`
2. Generates `keywords` from file path, title, and description when missing
3. Adds canonical Open Graph metadata using the site-level asset manifest
4. Normalizes legacy `og:image` values to section-level or fallback raster assets

**Usage:**
```bash
# Dry run (preview changes without modifying files)
npm run generate-seo:dry-run

# Apply changes
npm run generate-seo

# Process a single file
node tools/scripts/snippets/generate-seo.js --file=v2/home/mission-control.mdx
```

**Run before this script:** `node tools/scripts/snippets/generate-og-images.js`

**What gets added:**

**Keywords:**
- Base keyword: `livepeer`
- Path-based: extracted from folder structure (e.g., `home`, `gateways`, `developers`)
- Title-based: meaningful words from the page title
- Description-based: top 3 meaningful words from the description

**Open Graph metadata:**
- `og:image`
- `og:image:alt`
- `og:image:type`
- `og:image:width`
- `og:image:height`

**Asset assignment:**
- docs.json-routable pages use a top-level tab image for their locale (for example, `/snippets/assets/site/og-image/en/orchestrators.png`)
- non-routable authored pages use `/snippets/assets/site/og-image/fallback.png`
- SVG, GitHub `blob` URLs, and external OG assets are not used by this workflow

**Example output:**
```yaml
---
title: Gateway Overview
description: Learn about Livepeer gateways and how they work
keywords:
  - livepeer
  - gateways
  - gateway
  - overview
  - learn
'og:image': /snippets/assets/site/og-image/en/orchestrators.png
'og:image:alt': Livepeer Docs social preview image for Orchestrators
'og:image:type': image/png
'og:image:width': 1200
'og:image:height': 630
---
```

**When to run:**
- After creating new documentation pages
- After updating section labels or OG image branding
- Before deploying to re-normalize canonical OG fields

**Notes:**
- Adds frontmatter to authored MDX files when needed
- Rewrites stale `og:image` values to the canonical section or fallback asset
- Generates up to 10 keywords per page

### generate-og-images.js

Generates the site-level PNG asset set and manifest consumed by `generate-seo.js` and quality validation.

**What it does:**
1. Builds one fallback social image and one section image per top-level docs tab per locale
2. Writes raster assets to `snippets/assets/site/og-image/`
3. Writes `snippets/assets/site/og-image/manifest.json` for validators and generators

**Usage:**
```bash
# Preview generated files
node tools/scripts/snippets/generate-og-images.js --dry-run

# Generate the full asset set
node tools/scripts/snippets/generate-og-images.js

# Generate one asset only
node tools/scripts/snippets/generate-og-images.js --only=en:orchestrators
```

### update-component-library.sh

Auto-generates the component library index page from the current `snippets/components/` folder structure.

**What it does:**
1. Scans `snippets/components/` directory structure
2. Generates a `<Tree>` component with all folders and files
3. Updates `snippets/snippetsWiki/componentLibrary/index.mdx`

**Usage:**
```bash
./tools/scripts/snippets/update-component-library.sh
```

**When to run:**
- After adding new components to `snippets/components/`
- After reorganizing the components folder structure
- After renaming or deleting component files

**Output:**
Updates the auto-generated section in `snippets/snippetsWiki/componentLibrary/index.mdx` between the `AUTO-GENERATED` comments.

## Data Fetching

### fetch-openapi-specs.sh

Fetches OpenAPI specification files from the livepeer/ai-runner repository.

**Usage:**
```bash
./tools/scripts/snippets/fetch-openapi-specs.sh
```

**Downloads to `ai/worker/api/`:**
- `openapi.yaml` - AI Runner API spec
- `gateway.openapi.yaml` - AI Gateway API spec

---

### fetch-external-docs.sh

Fetches external documentation files from other Livepeer repositories and sanitizes them for MDX compatibility.

**Usage:**
```bash
./tools/scripts/snippets/fetch-external-docs.sh
```

**Downloads to `snippets/external/`:**
- `wiki-readme.mdx` - livepeer/wiki README
- `awesome-livepeer-readme.mdx` - livepeer/awesome-livepeer README
- `whitepaper.mdx` - Livepeer Whitepaper
- `gwid-readme.mdx` - videoDAC/livepeer-gateway README
- `box-additional-config.mdx` - go-livepeer box configuration

**Sanitization includes:**
- Escaping curly braces for MDX
- Removing HTML comments
- Converting HTML tags to Markdown equivalents

---

### fetch-lpt-exchanges.sh

Fetches LPT exchange listings from CoinGecko API and updates the exchanges page.

**Usage:**
```bash
./tools/scripts/snippets/fetch-lpt-exchanges.sh
```

**What it does:**
1. Fetches live data from CoinGecko API for Livepeer token
2. Generates a styled table of CEX exchanges with volume and trust scores
3. Appends DEX information and contract addresses
4. Updates `v2/pages/06_delegators/token-resources/lpt-exchanges.mdx`

**When to run:**
- Periodically to update exchange listings
- Before major releases to ensure current data
