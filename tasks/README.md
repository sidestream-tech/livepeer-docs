# tasks/

`tasks/` is a governed work artifact space. Every file here has a declared type. Files without a type do not belong here.

---

## Directory map

```
tasks/
├── plan/
│   ├── active/     ← in-flight plans (human or agent authored)
│   └── complete/   ← finished plans awaiting auto-deletion (30-day TTL)
├── reports/        ← script-generated outputs only
├── scripts/        ← temporary agent scripts only (14-day TTL)
└── errors/         ← active remediation notes tied to open issues or PRs
```

---

## Folder rules

### `tasks/plan/`

Human- and agent-authored planning documents. Every file in `plan/active/` must declare its `plan-type` in YAML frontmatter.

#### Plan frontmatter schema

```yaml
---
plan-type: agent-brief          # Required. See classification below.
retention: permanent            # Optional. Prevents all automated pruning.
tracked-issue: #123             # Auto-added by retention action. Do not set manually.
---
```

#### Plan classification

| `plan-type` | What it is | Copilot auto-assigned? |
|---|---|---|
| `agent-brief` | Executable task spec with clear steps and defined outputs. Ready to run. | ✅ Yes |
| `backlog` | Queued work, not yet started, no active owner. | ✅ Yes |
| `execution` | Active multi-step plan being worked across sessions, human-led. | ❌ No |
| `analysis` | Research output, audit plan, scoring analysis — human decision required. | ❌ No |
| `handoff` | Session context transfer or status summary. Informational only. | ❌ No |

- `plan/active/` — in-flight work. No age limit. Files unchanged >30 days trigger a GitHub issue.
- `plan/complete/` — completed work. Auto-deleted 30 days after last commit.
- `retention: permanent` — exempt from the stale-flagging action. Must be reviewed by a human quarterly.

#### Lifecycle

1. Plan is created in `plan/active/` with `plan-type` set.
2. Retention action opens a GitHub issue after 30 days of inactivity and tags the file with `tracked-issue: #NNN`.
3. For `agent-brief` and `backlog` types, the issue is auto-assigned to Copilot.
4. For other types, the issue is labelled `needs-human`.
5. When work is complete: move file to `plan/complete/`, close the issue. Auto-deleted after 30 more days.
6. To block auto-pruning: add `retention: permanent` to frontmatter.

#### GitHub issue labels used by retention action

| Label | Meaning |
|---|---|
| `tasks-retention` | Opened by the automated retention workflow |
| `plan-stale` | File unchanged >30 days |
| `agent-brief` / `backlog` / `handoff` / `analysis` / `execution` | Matches `plan-type` |
| `copilot-candidate` | Copilot has been auto-assigned |
| `needs-human` | Requires human review before action |
| `uncategorised` | No `plan-type` frontmatter found |

### `tasks/reports/`

Script-generated outputs only. Humans and agents do not write directly into this folder.

**Rules:**
- Every output path must be registered in the [Approved Generators](#approved-generators) table below.
- Outputs overwrite in place. One current copy per approved output path.
- No dated subdirectories. No one-off run directories. No ad hoc outputs.
- Seed files and baselines do not belong here — use governed config paths under `tools/config/`.
- `_legacy-unmanaged/` must contain no tracked files.

**Report metadata:**
All script-generated reports must include a header block identifying the generator script, run timestamp, and branch. This is enforced by the script governance framework.

**Report index:**
`tasks/reports/INDEX.md` is auto-generated by `audit-scripts.js` and lists all active output paths with their generators. Do not edit manually.

---

### `tasks/scripts/`

Temporary agent-authored scripts only. Scratch space for scripts generated mid-session that have not yet been migrated to `tools/scripts/`.

- **14-day hard TTL.** The retention action deletes all files older than 14 days, no exceptions.
- Exception: `audit-python.py` is preserved (actively wired script).
- New maintained scripts belong in `tools/scripts/`, not here.

---

### `tasks/errors/`

Active remediation notes tied to open issues or PRs.

- Each file should reference an open GitHub issue or active PR.
- Files unchanged for 14 days trigger a GitHub issue flagging them for review.
- Resolved errors are deleted, not archived.

## Protected data

`tasks/reports/script-classifications.json` is canonical governance data for the script governance framework. **Never move, rename, or delete it.**

---

## Retention enforcement

Workflow: `.github/workflows/tasks-retention.yml` — runs weekly on `docs-v2`, Mondays 08:00 UTC.

| Target | Behaviour |
|---|---|
| `tasks/plan/complete/**` | Hard delete files with last commit >30 days old |
| `tasks/plan/active/**` | Open GitHub issue + tag file with issue number if unchanged >30 days |
| `tasks/scripts/**` | Hard delete files >14 days old (except `audit-python.py`) |
| `tasks/errors/**` | Open GitHub issue if unchanged >14 days |
| `tasks/reports/**` | No action — scripts manage their own overwrite cycle |
| `tasks/reports/script-classifications.json` | Never touched by any automated process |

**Required labels** (must exist in repo before first workflow run):
`tasks-retention`, `plan-stale`, `error-stale`, `agent-brief`, `backlog`, `handoff`, `analysis`, `execution`, `copilot-candidate`, `needs-human`, `uncategorised`

---

## Naming

- Lowercase kebab-case for all directories and new filenames.
- No spaces. No uppercase.
- Legacy basenames (`LINK_TEST_REPORT.*`, `SCRIPT_AUDIT.*`) normalised when generators are next modified.

---

## Change control

1. Do not add a new report output path without adding it to the Approved Generators table.
2. Do not commit manual edits, research, or context dumps into `tasks/reports/`.
3. Do not create subdirectories in `tasks/reports/` without a corresponding registered generator.
4. `tasks/context_data/` is fully deprecated. Contents move to the `docs-v2-assets` branch.
5. Do not set `tracked-issue` manually — written by the retention action only.
