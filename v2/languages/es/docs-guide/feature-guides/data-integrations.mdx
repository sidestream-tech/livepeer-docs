## Especificaciones de la API

Archivos de especificación de API canónicos en `api/`:

- `api/openapi.yaml`
- `api/openapi.json`
- `api/studio.yaml`
- `api/gateway.openapi.yaml`
- `api/ai-worker.yaml`
- `api/cli-http.yaml`

Ayudas para generación relacionadas:

- `tools/scripts/snippets/fetch-openapi-specs.sh`
- `tools/scripts/snippets/generate-api-docs.sh`

## Fuentes de datos externas automatizadas

Salidas de alimentación externa principal:

- Foro -> `snippets/automations/forum/forumData.jsx`
- Blog Ghost -> `snippets/automations/blog/ghostBlogData.jsx`
- YouTube -> `snippets/automations/youtube/youtubeData.jsx`
- Lanzamiento/estado global -> `snippets/automations/globals/globals.mdx`
- Mostrar -> `snippets/automations/showcase/showcaseData.jsx`

Definiciones de pipeline:

- Flujos de trabajo de GitHub bajo `.github/workflows/update-*.yml`
- Activos de n8n bajo `snippets/automations/scripts/n8n/`

## Capas de datos internas

- Mapas de ruta/enlace: `snippets/data/*/hrefs.jsx`
- Módulos de datos de dominio: `snippets/data/**/*.jsx`
- Fragmentos de documentos de variables/meta: `snippets/data/variables*.mdx`
- Mapa del sitio de IA: `sitemap-ai.xml` (generado por `tools/scripts/generate-ai-sitemap.js`, actualizado mediante `.github/workflows/generate-ai-sitemap.yml`)

## Integraciones mencionadas en documentos de producto

El repositorio incluye documentos orientados a integraciones para:

- LPT y superficies de datos de intercambio/ referencia relacionados
- Integraciones del explorador y el protocolo/recursos
- Resúmenes de GitHub y ecosistema mediante salidas de automatización

Utilice las páginas de documentación pública para detalles orientados al usuario y este archivo para asignación de ubicación para mantenimiento.

## Notas operativas

- Trate los artefactos de datos generados como estado derivado.
- Actualice los scripts/flujos de trabajo de la pipeline, luego regenere los artefactos e índices.
- Prefiera salidas deterministas y rutas de escritura estables para previsibilidad en CI.
