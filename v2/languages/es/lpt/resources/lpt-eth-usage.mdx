### Capas de pago

El protocolo Livepeer utiliza dos monedas distintas para diferentes propósitos:

| Moneda | Propósito                            | Usado por                   |
| -------- | ---------------------------------- | ------------------------- |
| ETH/Wei  | Pagos de servicio (transcodificación, IA) | Puertas de enlace → Orquestadores  |
| LPT      | Staking, gobernanza, recompensas       | Orquestadores, Delegadores |

La separación es intencional: ETH maneja los pagos reales de servicio, mientras que LPT maneja la gobernanza del protocolo y el staking.
Este diseño mantiene los costos de servicio predecibles en una moneda estable, mientras permite que LPT cumpla su función de gobernanza

**Configuración de precios**Los precios se pueden especificar en wei o convertidos de divisas fiduciarias[starter.go:891-908](https://github.com/livepeer/go-livepeer/blob/5691cb48/cmd/livepeer/starter.go#L891-L908)

```go
pricePerUnit, currency, err := parsePricePerUnit(*cfg.PricePerUnit)
// Supports: "1000", "0.50USD", "1.5EUR" etc.
// Converts to wei for actual payments
```

## ETH Pagos

Los gateways pagan por los servicios de transcodificación y procesamiento de IA utilizando ETH (Ethereum), no tokens Livepeer (LPT). El sistema de pago se basa en la moneda de Ethereum.
