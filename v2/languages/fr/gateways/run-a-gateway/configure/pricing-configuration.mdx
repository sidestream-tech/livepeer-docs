Les passerelles dans Livepeer configurent les prix pour contrôler les coûts et les marges.
En tant que consommateurs de services de transcodage vidéo et d'IA, les passerelles définissent les prix maximum
qu'elles sont prêtes à payer aux Orchestrateurs pour le travail de traitement.

## Concepts de tarification

#### Devise de paiement

Tous les paiements sont effectués en <Badge color="green">ETH(wei)</Badge> - **pas** Livepeer jetons (LPT).

#### Modèles de tarification

<Warning> Format me.</Warning>

1. <Badge color="blue">Video Transcoding</Badge>: Par segment vidéo traité

- Unité : Pixels traités (largeur x hauteur)
- Calcul : pixels traités × prix par pixel

2. <Badge color="purple"> AI Processing</Badge>: Tarifé par capacité /
modèle.{' '}

- Unité : Prix par capacité/modèle
- Calcul :
  - Paiements par pixel : Calculés comme largeur × hauteur × sorties
  - Paiements par requête : Paiement unique par requête d'IA
  - Paiements vidéo en direct : Paiements basés sur des intervalles pendant le streaming (configurable)

## Calcul et processus de paiement des frais

<Warning> Format me. Add diagram </Warning>

Les passerelles paient les frais via différents mécanismes selon le type de charge de travail :

_Transcodage vidéo_

- Paiements par segment : Chaque segment vidéo génère un ticket de paiement -> [<Icon icon="github" /> segment_rpc.go :](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/segment_rpc.go)
- Calcul des frais : Basé sur les pixels traités × prix par pixel -> [<Icon icon="github" /> segment_rpc.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/segment_rpc.go)

_Traitement IA_

- Paiements par pixel : Calculés comme largeur × hauteur × sorties [<Icon icon="github" /> live_payment.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/live_payment.go)
- Vidéo en direct : Utilise des paiements basés sur des intervalles pendant le streaming [<Icon icon="github" /> ai_http.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/server/ai_http.go)

_Flux de traitement des paiements_

1. La passerelle envoie le paiement avec le segment/requête à l'orchestrateur [<Icon icon="github" /> live_payment.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/live_payment.go)
2. L'orchestrateur valide le paiement et met à jour le solde [<Icon icon="github" /> segment_rpc.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/segment_rpc.go)
3. Les frais sont débités du compte de solde de la passerelle [<Icon icon="github" /> ai_http.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/server/ai_http.go)

## Méthodes de configuration

<Warning> Format me. </Warning>
Les passerelles définissent les prix maximum qu'elles sont prêtes à payer via des drapeaux de configuration dans
le `transcodingConfig.json` ou directement dans la CLI.

1. **Ligne de commande** drapeaux
   ```bash
     -maxPricePerUnit=1000 \
     -pixelsPerUnit=1
   ```
2. **Configuration JSON** fichier (format texte brut clé-valeur)

   Pour les capacités IA, utilisez des fichiers JSON avec une tarification spécifique au modèle

3. **API HTTP** - peut être utilisée à l'exécution pour effectuer des ajustements sans redémarrage
   - /setBroadcastConfig : Définir la tarification générale
   - /setMaxPriceForCapability: Définir le prix du modèle IA
4. **Outil CLI**

   Utilisez livepeer_cli -> **Option 16**: "Définir la configuration de diffusion"

## Configuration de l'Orchestrateur et informations tarifaires (Référence de la passerelle)

Une référence pour les opérateurs de passerelle sur la façon dont les orchestrateurs configurent les tarifs et les frais des services.

#### Tarification par passerelle (peut être définie par les orchestrateurs)

Les orchestrateurs peuvent définir des prix spécifiques pour chaque passerelle en utilisant
`-pricePerGateway` [<Icon icon="github" /> starter.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/cmd/livepeer/starter.go)

```json
{
  "gateways": [
    {
      "ethaddress": "0x123...",
      "priceperunit": 0.5,
      "currency": "USD",
      "pixelsperunit": 1000000000000
    }
  ]
}
```

#### Calcul du prix

<Warning> Repeated </Warning>

Le calcul réel du prix se produit dans le fichier de l'orchestrateur `priceInfo` fonction [<Icon icon="github" /> orchestrator.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/orchestrator.go)

1. Vérifie les prix fixes par ID de manifeste
2. Récupère le prix de base à partir de la configuration de l'orchestrateur
3. Pour les charges de travail IA, additionne les prix des paires capacité/modèle individuelles
4. Applique un ajustement automatique basé sur les coûts de transaction si activé

#### Ajustement dynamique du prix

Les orchestrateurs peuvent activer les ajustements automatiques des prix basés sur les coûts de transaction – c'est pourquoi il est important de définir les drapeaux maxPricing -> [<Icon icon="github" /> orchestrator.go](https://github.com/livepeer/go-livepeer/blob/5691cb48/core/orchestrator.go)

```go
if !orch.node.AutoAdjustPrice {
    return basePrice, nil
}
// Apply overhead multiplier based on tx costs
overhead := big.NewRat(1, 1)
if basePrice.Num().Cmp(big.NewInt(0)) > 0 {
    txCostMultiplier, err := orch.node.Recipient.TxCostMultiplier(sender)
    if txCostMultiplier.Cmp(big.NewRat(0, 1)) > 0 {
        overhead = overhead.Add(overhead, new(big.Rat).Inv(txCostMultiplier))
    }
}
```

#### Notes

- **Drapeaux de passerelle** contrôlent ce que vous **payez** (prix max)
- **Drapeaux d'orchestrateur** contrôlent ce que vous **facturez** (prix réels)
- La tarification IA utilise le `maxPricePerCapability` structure JSON pour la tarification par modèle
- Tous les prix peuvent être spécifiés en wei ou avec un suffixe de devise (par ex., "0.50USD")
- Les valeurs par défaut "0" signifient accepter n'importe quel prix ou utiliser les paramètres par défaut du système

Gateways in Livepeer set prices and pay fees through a multi-layered system that includes
price limits, per-gateway pricing, and payment processing for both video transcoding and AI workloads.

Gateways use a probabilistic micropayment system where they generate payment tickets for work done.
The system supports both traditional video transcoding (per segment) and AI processing (per pixel)
with different pricing models but unified payment infrastructure.
