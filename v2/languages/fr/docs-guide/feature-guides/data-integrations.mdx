## Spécifications de l'API

Fichiers API/spécifications canoniques dans `api/`:

- `api/openapi.yaml`
- `api/openapi.json`
- `api/studio.yaml`
- `api/gateway.openapi.yaml`
- `api/ai-worker.yaml`
- `api/cli-http.yaml`

Aideurs pour la génération associée :

- `tools/scripts/snippets/fetch-openapi-specs.sh`
- `tools/scripts/snippets/generate-api-docs.sh`

## Alimentations externes automatisées

Sorties d'alimentation externe principales :

- Forum -> `snippets/automations/forum/forumData.jsx`
- Blog Ghost -> `snippets/automations/blog/ghostBlogData.jsx`
- YouTube -> `snippets/automations/youtube/youtubeData.jsx`
- Version de sortie/état global -> `snippets/automations/globals/globals.mdx`
- Exposition -> `snippets/automations/showcase/showcaseData.jsx`

Définitions de pipeline :

- Flux GitHub sous `.github/workflows/update-*.yml`
- actifs n8n sous `snippets/automations/scripts/n8n/`

## Couches de données internes

- Cartes de route/liens : `snippets/data/*/hrefs.jsx`
- Modules de données de domaine : `snippets/data/**/*.jsx`
- Extraits de variables/méta-documents : `snippets/data/variables*.mdx`
- Plan du site AI : `sitemap-ai.xml` (généré par `tools/scripts/generate-ai-sitemap.js`, actualisé via `.github/workflows/generate-ai-sitemap.yml`)

## Intégrations mentionnées dans les docs produit

Le dépôt inclut des docs orientés intégration pour :

- LPT et surfaces de données d'échange/références associées
- Intégrations de l'explorateur et du protocole/resources
- Résumés GitHub et de l'écosystème via des sorties d'automatisation

Utilisez les pages de docs publiques pour les détails destinés aux utilisateurs et ce fichier pour la cartographie de l'emplacement des mainteneurs.

## Remarques opérationnelles

- Traitez les artefacts de données générés comme un état dérivé.
- Mettez à jour les scripts/travaux, puis régénérez les artefacts et les index.
- Privilégiez les sorties déterministes et les chemins d'écriture stables pour une prévisibilité CI.
