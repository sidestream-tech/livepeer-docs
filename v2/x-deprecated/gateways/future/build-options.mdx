## Downloading Pre-built Binaries (Recommended)

### From GitHub Releases
Binaries are built and uploaded for each release:
```bash
# Download the latest release for your platform
wget https://github.com/livepeer/go-livepeer/releases/download/v0.5.32/livepeer-linux-amd64.tar.gz
tar -xzf livepeer-linux-amd64.tar.gz
```

### From Docker Hub
Pull the pre-built Docker image:
```bash
docker pull livepeer/go-livepeer:master
# Or for a specific version
docker pull livepeer/go-livepeer:v0.5.32
```

## When to Build From Source

Only build from source if you need to:
- Modify the code
- Test unreleased features
- Build for a specific configuration
- Debug issues

## Binary Release Process

The CI/CD pipeline automatically builds binaries for:
- Linux (amd64, arm64) [1](#63-0) 
- macOS (amd64, arm64) [2](#63-1) 
- Windows (amd64) [3](#63-2) 

These are uploaded to GitHub releases and Docker Hub with each tagged version [4](#63-3) .

## Notes

- Pre-built binaries include all dependencies and are ready to run
- Docker images are built for linux/amd64 only [5](#63-4) 
- Source builds allow custom build tags: `BUILD_TAGS=mainnet,experimental make` [6](#63-5) 
- The master branch builds are compatible with mainnet contracts [7](#63-6)

#### master

This branch is compatible with contracts on Rinkeby and mainnet. Code committed to this branch MUST NOT break contract compatibility on Rinkeby or mainnet.
