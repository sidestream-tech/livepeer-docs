## Visión general

Livepeer está actualmente desplegado en la [Arbitrum](https://docs.arbitrum.io/build-decentralized-apps/public-chains#arbitrum-one) red L2 (Arbitrum One).

Para interactuar con el [Livepeer contratos inteligentes](../../../../resources/references/contract-addresses) necesitarás conectarte a un RPC Arbitrum.

Si planeas ejecutar tu Gateway en **En cadena**modo, necesitarás:

- Una cuenta de Ethereum con fondos para pagar las tarifas de transacción
- Una URL RPC de Ethereum para conectarse a la red Arbitrum donde se despliega Livepeer

## Notas de seguridad

<Warning>
  - Never share your `keystore` files - they contain your encrypted private keys
  - Use strong passwords for `keystore` encryption - Back up your `keystore`
  securely - The `ethPassword` only protects the `keystore` file - your actual
  private key is stored encrypted within the keystore - Never share your wallet
  private key. In case of accidental public exposure - remove any funds &
  change your wallet immediately
</Warning>

    Livepeer creará automáticamente una nueva billetera y almacén de claves para ti si no proporcionas una dirección de billetera existente después del primer [instalar tu software Gateway](/v2/es/gateways/run-a-gateway/install/install-overview).

    El software detectará que no existe un almacén de claves para esa dirección y te pedirá que:
        - Introduce tu clave privada
        - Crea una dirección de cuenta (billetera) `-ethAcctAddr`
        - Establece una contraseña para el cifrado del almacén de claves `-ethPassword`
        - Crea el archivo keystore `-ethKeystorePath`

    Utiliza el siguiente comando para iniciar rápidamente una pasarela on-chain con una nueva billetera y generar automáticamente todos los parámetros requeridos.

    Ver [RPC URL](#rpc-url) para obtener instrucciones sobre cómo encontrar o usar una URL RPC alternativa Arbitrum.

    ```bash icon="terminal" wrap lines Create New Wallet & Keystore
    livepeer -gateway \
    -network arbitrum-one-mainnet \
    -ethUrl=https://arb1.arbitrum.io/rpc #public RPC: rate-limited
    \ -ethAcctAddr="" \
    -ethKeystorePath="" \
    -ethPassword=""
    ```

    Si deseas usar una billetera existente, coloca la dirección pública en el `-ethAcctAddr` indicador.

    ```bash icon="terminal" wrap lines Exisiting Wallet, New Keystore
    livepeer -gateway \
        -network arbitrum-one-mainnet \
        -ethUrl=https://arb1.arbitrum.io/rpc #public rate-limited RPC
        \ -ethAcctAddr=<YOUR_WALLET_ADDRESS> \
        -ethKeystorePath=""
    ```

    To bring your existing Ethereum account to Livepeer, you need to create a keystore file
    with your private key and place it in the correct directory structure.

    The keystore files follow Ethereum's UTC-timestamp-address format.
    These files, often referred to as "UTC / JSON Wallets" or "Web3 Secret Storage Definition" files,
    are encrypted local files that contain your Ethereum private key, allowing access to your account.

    Livepeer uses standard Ethereum keystore encryption, so any tool that creates Ethereum keystores will work.
    Create a keystore file using Livepeer devtools or Ethereum tooling.

    ### Create Automatically with Livepeer Tooling
    <Tip>
        It's recommended to let Livepeer devtools automatically create the keystore on first run after [installing the Gateway software](/v2/es/gateways/run-a-gateway/install/install-overview).
