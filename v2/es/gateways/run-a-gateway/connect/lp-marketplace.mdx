# ¿Qué es el Marketplace?

El Livepeer Marketplace permite**_Orchestrators_** publicar la disponibilidad de cálculo,
servicios de cálculo (IA y video), precios y características de rendimiento.

El Marketplace es**NO**un mercado público para los usuarios finales, sino que es una interfaz programática para que los Gateways descubran y conecten aplicaciones a servicios disponibles.

En el Mercado Livepeer,**los Gateways actúan como un motor de búsqueda**

Descubren ofertas de Orchesters, aplican lógica de selección y enrutamiento, y gestionan el acceso a servicios adecuados para las aplicaciones según requisitos de costo, rendimiento y política.

<br />
> - **Orchesters**publican servicios en el Mercado Livepeer; > - **Puertas de enlace**
filtrar y conectar las solicitudes de la aplicación a esos servicios.

## Detalles de la oferta del orquestador

Una oferta es una declaración estructurada de:

- Modelos compatibles (por ejemplo, SDXL, ControlNet, modelos de profundidad)
- Pipelines compatibles (ComfyStream, Daydream, BYOC)
- Precios por cuadro / por segundo / por solicitud
- Niveles de GPU y métricas de rendimiento
- Disponibilidad regional
- SLAs y latencia esperada

Los orquestadores publican sus ofertas a través de la`OrchestratorInfo`estructura de datos

```protobuf
message OrchestratorInfo {
    string transcoder = 1;           // Service URI
    TicketParams ticket_params = 2;   // Payment parameters
    PriceInfo price_info = 3;         // Pricing information
    bytes address = 4;               // ETH address
    Capabilities capabilities = 5;    // Supported features
    AuthToken auth_token = 6;         // Authentication
    repeated HardwareInformation hardware = 7;  // Hardware specs
    repeated OSInfo storage = 32;     // Storage options
    repeated PriceInfo capabilities_prices = 33;  // AI model pricing
}
````

Los orquestadores publican ofertas a nivel de cálculo en el Marketplace para que los Gateways las descubran y enruten trabajos a ellas.
