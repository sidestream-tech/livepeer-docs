Studio Livepeer **API de transcodificación** te permite transcodificar archivos de video que se almacenan en **almacenamiento compatible con S3**(por ejemplo, Storj, W3S, AWS S3). Especifica una ubicación de entrada (bucket/ruta) y ubicaciones de salida para HLS y/o MP4; Livepeer transcodifica y escribe los resultados en tu almacenamiento.

<Info>
  The Transcode API works with any S3-compatible provider. You need valid S3 credentials (access key, secret key, endpoint, bucket) for both the source and destination.
</Info>

## Requisitos previos

- Un [Livepeer Studio cuenta](https://livepeer.studio) y [clave de API](../reference/api).
- Archivo de video ya subido a su cubo compatible con S3 (por ejemplo, mediante Storj uplink, CLI de AWS o SDK de S3).
- Credenciales de S3 con acceso de lectura a la ruta de origen y acceso de escritura a la ruta de salida.

## Flujo básico

1. **Obtener credenciales de S3** para su proveedor (por ejemplo.[Credenciales de S3 de Storj](https://docs.storj.io/dcs/api-reference/s3-compatible-gateway).)
2. **Llame a la API de transcodificación** con:
   - `input`: tipo `s3`, endpoint, credenciales, bucket, ruta al video de origen.
   - `storage`: tipo `s3`, endpoint, credenciales, bucket (y prefijo de ruta opcional para salidas).
   - `outputs`: por ejemplo, `hls: { path: "/output/hls" }`, `mp4: { path: "/output/mp4" }`.
3. Consulte la tarea devuelta o escuche webhooks hasta que el trabajo se complete. Los archivos de salida aparecen en su cubo en las rutas proporcionadas.

Para ver la forma exacta de la solicitud y la respuesta, consulte el [referencia de la API de transcodificación](https://livepeer.studio/docs/api-reference/transcode/create). Los ejemplos de código para Node.js y otros lenguajes están disponibles en los documentos de [Livepeer Studio docs](https://livepeer.studio/docs).
