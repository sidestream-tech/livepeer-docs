Después de [instalar go-livepeer](/v2/orchestrators/setup/install-go-livepeer), configura el nodo para que los gateways puedan encontrarte y enviar trabajo. Esta página cubre las banderas principales y los archivos de configuración opcionales.

## Configuración principal

### Red y billetera

- **-ethUrl** - Arbitrum URL de RPC. Consulte [Conéctese a Arbitrum](/v2/orchestrators/setup/connect-to-arbitrum).
- **-ethAcctAddr** - (Opcional) dirección de cuenta ETH si ya tiene un archivo keystore.
- **-network** - Usar `arbitrum-one-mainnet` para producción.

### Identidad del orquestador y precios

- **-serviceAddr** - Nombre de host público o IP y puerto (por ejemplo: `orchestrator.example.com:8935`). Preferir un nombre de host que controle; cambiarlo más tarde requiere una actualización en cadena.
- **-pricePerUnit** - Precio en wei por píxel (transcodificación). Obligatorio al iniciar; puede actualizarse más tarde mediante la CLI o en cadena.
- **-pixelsPerUnit** - Píxeles por unidad de precio (por defecto 1). Afecta cómo se calcula el precio por segmento.

### GPU y modo

- **-orchestrator** - Ejecutar en modo orquestador.
- **-transcodificador** - Habilitar transcodificación con GPU.
- **-nvidia** - Identificadores de GPU NVIDIA separados por comas (por ejemplo, `0,1`). Omitir para uso solo con CPU.

Ejemplo de ejecución mínima:

<CustomCodeBlock
  codeString={`livepeer \\
  -orchestrator \\
  -transcoder \\
  -network arbitrum-one-mainnet \\
  -ethUrl <ETH_URL> \\
  -serviceAddr <HOST:8935> \\
  -pricePerUnit <WEI_PER_PIXEL> \\
  -nvidia 0,1`}
  language="bash"
  icon="terminal"
/>

## Opcional: opciones de transcodificación

Puedes definir perfiles de transcodificación en un archivo JSON y referenciarlos. Estructura de ejemplo:

## Opcional: modelos de IA (orquestadores de IA)

Si ejecutas inferencia de IA, crea un**aiModels.json** para especificar pipelines y modelos:

<CustomCodeBlock
  codeString={`[
  {
    "pipeline": "text-to-image",
    "model_id": "stabilityai/sd-turbo",
    "warm": true
  }
]`}
  language="json"
  icon="brackets-curly"
  filename="aiModels.json"
/>

Ver [pipelines de IA](/v2/orchestrators/advanced/ai-pipelines) y [Desarrolladores BYOC](/v2/developers/ai-pipelines/byoc) para configuración completa de IA.

## Verificación

Después de iniciar el nodo:

- Los registros muestran`Listening for RPC on :8935` y no hay errores de RPC.
- [Explorador](https://explorer.livepeer.org/) muestra su nodo después de la activación.
- Bajo carga, el uso de la GPU (por ejemplo, `nvidia-smi`) aumenta cuando se procesan trabajos.
