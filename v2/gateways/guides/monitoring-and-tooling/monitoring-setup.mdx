This guide sets up production observability for your Livepeer gateway. It assumes your gateway is already running — if you need quick health verification first, see [Gateway Health Checks](/v2/gateways/guides/monitoring-and-tooling/gateway-health-checks).

For on-chain gateways, Prometheus metrics complement but do not replace [on-chain debugging](/v2/gateways/guides/monitoring-and-tooling/on-chain-debugging) — contract events and deposit state require separate monitoring.

## Step 1 — Enable Prometheus Metrics

Add the `-monitor` flag when starting your gateway. Without it, the `/metrics` endpoint is not exposed.

```bash
# Video gateway with monitoring enabled
livepeer \
  -gateway \
  -network arbitrum-one-mainnet \
  -ethUrl https://arb1.arbitrum.io/rpc \
  -monitor \
  -metricsPerStream \
  [other flags...]

# AI gateway with monitoring enabled
livepeer \
  -gateway \
  -monitor \
  -httpAddr 0.0.0.0:8937 \
  -orchAddr https://orch1.example.com:8936 \
  [other flags...]
```

**Monitoring flags:**

| Flag | Purpose | Recommendation |
|------|---------|----------------|
| `-monitor` | Enables the `/metrics` Prometheus endpoint | Always enable in production |
| `-metricsPerStream` | Groups performance metrics by individual stream | Enable for per-stream debugging; note higher metric cardinality |
| `-metricsClientIP` | Includes client IP as a metric label | Enable only if you need per-client breakdown |

**Verify metrics are exposed:**

```bash
curl http://localhost:8935/metrics | head -30
```

You should see Prometheus-format output with `livepeer_` prefixed metric names.

## Step 2 — Prometheus Configuration

Create or update your `prometheus.yml` to scrape the gateway:

```yaml
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'livepeer-video-gateway'
    static_configs:
      - targets: ['localhost:8935']
    metrics_path: '/metrics'
    scrape_interval: 15s
    scrape_timeout: 10s
    labels:
      gateway_type: 'video'
      environment: 'production'

  - job_name: 'livepeer-ai-gateway'
    static_configs:
      - targets: ['localhost:8937']
    metrics_path: '/metrics'
    scrape_interval: 15s
    labels:
      gateway_type: 'ai'
      environment: 'production'
```

For a dual gateway, include both scrape jobs — they expose different metrics from the same node.

## Key Metrics — Video Gateway

These are the metrics that matter most for on-chain video gateways. Full reference: [Prometheus Metrics](/v2/gateways/resources/technical/go-livepeer/prometheus-metrics).

### Health and Performance

| Metric | What it tells you | Alert threshold |
|--------|------------------|-----------------|
| `livepeer_success_rate` | Transcoded segments ÷ source segments | Alert when `< 0.95` |
| `livepeer_transcode_overall_latency_seconds` | End-to-end transcoding latency | Alert when `> 5s` |
| `livepeer_segment_transcode_failed_total` | Cumulative failed transcode count | Alert on rate increase |
| `livepeer_http_client_segment_transcoded_realtime_slow` | Segments that took 2× longer than realtime | Alert on sustained increase |
| `livepeer_orchestrator_swaps` | Mid-stream orchestrator changes | Alert on high rate |
| `livepeer_current_sessions_total` | Active transcoding sessions | Capacity planning |

### Payment Metrics

| Metric | What it tells you | Alert threshold |
|--------|------------------|-----------------|
| `livepeer_gateway_deposit` | Remaining ETH deposit (wei) | Alert when `< 10000000000000000` (0.01 ETH) |
| `livepeer_gateway_reserve` | Remaining ETH reserve (wei) | Alert when `< 5000000000000000` (0.005 ETH) |
| `livepeer_ticket_value_sent` | ETH value of tickets issued | Budget tracking |
| `livepeer_tickets_sent` | Total tickets issued | Volume tracking |
| `livepeer_payment_create_errors` | Failed ticket creation | Alert on any increase |
| `livepeer_suggested_gas_price` | Arbitrum gas price oracle | Alert on spike (affects orchestrator costs) |

<Warning>
  `livepeer_gateway_deposit` and `livepeer_gateway_reserve` are your most critical alerts. When deposit hits zero, job routing stops immediately with no warning to end users.
</Warning>

### Useful Prometheus Queries

```promql
# Success rate over last 5 minutes
rate(livepeer_segment_transcoded_total[5m]) /
  rate(livepeer_segment_source_appeared_total[5m])

# ETH deposit remaining in ETH (not wei)
livepeer_gateway_deposit / 1e18

# Failed transcode rate per minute
rate(livepeer_segment_transcode_failed_total[1m])

# Average transcoding latency
rate(livepeer_transcode_overall_latency_seconds_sum[5m]) /
  rate(livepeer_transcode_overall_latency_seconds_count[5m])
```

## Key Metrics — AI Gateway

AI gateways operate off-chain. There are fewer built-in metrics than video gateways, but the ones that exist are critical.

| Metric | What it tells you | Alert threshold |
|--------|------------------|-----------------|
| `livepeer_current_sessions_total` | Active AI sessions | Capacity planning |
| `livepeer_max_sessions_total` | Configured session limit | Alert when `current > 0.8 × max` |
| `livepeer_discovery_errors_total` | Orchestrator discovery failures | Alert on any increase |
| `livepeer_segment_source_upload_failed_total` | Failed job submissions | Alert on rate increase |

<Note>
  AI-specific inference latency (per-pipeline, per-model) is not currently exposed via `/metrics`. For AI inference latency monitoring, use [Livepeer Tools](/v2/gateways/guides/monitoring-and-tooling/tools-and-dashboards#livepeer-tools-dashboard) or instrument your own application layer.

# Sessions approaching capacity

## Key Metrics — Dual Gateway

Run both video and AI metric sets. The additional concern for dual gateways is GPU resource contention.

**GPU monitoring via HTTP API:**

```bash
# Poll hardware stats every 30 seconds
watch -n 30 'curl -s http://localhost:8935/hardware/stats | python3 -m json.tool'
```

Key fields to watch:
- `gpu_memory_used` / `gpu_memory_total` — approach to 100% precedes model load failures
- `gpu_utilization` — sustained 100% during low-volume periods indicates bottleneck

**When to separate workloads:**

Consider splitting video and AI onto separate nodes when:
- GPU memory is consistently above 85%
- AI model loading causes video transcoding latency spikes
- `livepeer_orchestrator_swaps` rate increases during AI inference peaks

## Docker Monitoring Stack

The Livepeer Foundation maintains a Docker monitoring container that bundles Prometheus, Grafana, and starter dashboard templates.

{/* REVIEW: Rick — confirm livepeer/livepeer-monitoring repo is current and maintained */}

```bash
# Clone the monitoring repository
git clone https://github.com/livepeer/livepeer-monitoring
cd livepeer-monitoring

# Start the stack
docker compose up -d
```

The stack starts:
- **Prometheus** on port `9090` — scraping your gateway at `localhost:8935`
- **Grafana** on port `3000` — default credentials `admin / admin`

Update `prometheus.yml` in the repo to point to your gateway host if it is not on `localhost`.

**Default Grafana dashboards included:**
- Node overview (sessions, success rate, latency)
- Payment overview (deposit, reserve, ticket value)
- Transcoding performance (realtime ratios, latency distribution)

## Alert Recommendations

### Critical Alerts — Act Immediately

```yaml
groups:
  - name: livepeer-gateway-critical
    rules:
      - alert: GatewayDown
        expr: up{job=~"livepeer.*"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "Livepeer gateway is unreachable"

      - alert: DepositExhausted
        expr: livepeer_gateway_deposit < 10000000000000000  # 0.01 ETH
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "Gateway ETH deposit critically low ({{ $value | humanize }} wei)"

      - alert: SuccessRateCritical
        expr: livepeer_success_rate < 0.90
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "Transcoding success rate below 90%"
```

### Warning Alerts — Investigate

```yaml
      - alert: DepositLow
        expr: livepeer_gateway_deposit < 50000000000000000  # 0.05 ETH
        for: 10m
        labels:
          severity: warning
        annotations:
          summary: "Gateway ETH deposit below 0.05 ETH — top up soon"

      - alert: SuccessRateWarning
        expr: livepeer_success_rate < 0.95
        for: 5m
        labels:
          severity: warning

      - alert: HighTranscodeLatency
        expr: |
          rate(livepeer_transcode_overall_latency_seconds_sum[5m]) /
          rate(livepeer_transcode_overall_latency_seconds_count[5m]) > 5
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Average transcoding latency exceeds 5 seconds"

      - alert: PaymentCreateErrors
        expr: rate(livepeer_payment_create_errors[5m]) > 0
        for: 2m
        labels:
          severity: warning
        annotations:
          summary: "Ticket creation errors — check Arbitrum RPC health"

      - alert: AIDiscoveryErrors
        expr: rate(livepeer_discovery_errors_total[5m]) > 0
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "Orchestrator discovery failing — check -orchAddr list"
```
