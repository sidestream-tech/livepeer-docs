## Objectifs

- fournir des parcours de compréhension du zéro au héros
- préserver la correction technique tout en restant axé sur le produit
- aligner le contenu avec les rôles utilisateurs réels et les objectifs opérationnels

## Modèle d'IA

Les principales zones de documentation sont orientées par le rôle et l'intention :

- Accueil / À propos pour l'orientation et les concepts de base
- Communauté pour la participation à l'écosystème
- Développeurs / Portails / Orchestrateurs / Délégués pour les workflows de rôle
- Ressources pour les références, les guides et l'outillage de documentation interne

Fichiers sources de navigation :

- `docs.json`
- `v2/pages/**`

## Couches de contenu

### Couche 1 : Produit et positionnement

- proposition de valeur et cadre de contexte
- points d'entrée du parcours utilisateur
- résumés concis orientés décisions

### Couche 2 : Comment faire opérationnellement

- installation, manuels d'exploitation, pages de migration/checklist
- flux de travail pratiques avec des résultats attendus

### Couche 3 : Référence approfondie

- APIs, documents de schéma/spécification, matrices de commandes, détails des composants
- dépannage et documentation des cas limites

## Texte + Principes de qualité

- maintenir la clarté du produit sans sacrifier la précision technique
- préférer les références canoniques aux textes procéduraux dupliqués
- assurer que les affirmations sont traçables jusqu'au code, à la configuration de workflow ou aux docs principales
- garder les docs maintenables grâce à l'automatisation et à la validation

## Docs de support canoniques

- pages de guide de documentation publique :
  - `v2/resources/documentation-guide/documentation-guide.mdx`
  - `v2/resources/documentation-guide/style-guide.mdx`
  - `v2/resources/documentation-guide/component-library.mdx`
- Navigation interne/gouvernance :
  - `docs-guide/overview.mdx`
  - `docs-guide/policies/source-of-truth-policy.mdx`

## Guides pour les mainteneurs

Lors de l'ajout de nouvelles surfaces de contenu :

1. Placer le contenu dans la bonne couche d'IA et le contexte de rôle.
2. Ajouter ou mettre à jour la navigation dans `docs.json` où cela est approprié.
3. Ajouter des hooks d'automatisation/génération d'index uniquement lorsqu'une répétabilité est claire.
4. Mettre à jour les cartes de documentation-guide afin que les mainteneurs puissent découvrir et gouverner la fonctionnalité.
