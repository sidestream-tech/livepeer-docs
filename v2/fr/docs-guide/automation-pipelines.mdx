## Surfaces d'automatisation

- Workflows GitHub Actions dans`.github/workflows/`
- Scripts d'aide pour les workflows dans`.github/scripts/`
- Actifs de pipeline n8n dans`snippets/automations/scripts/n8n/`
- Scripts de maintenance dans`tools/scripts/`

Voir l'inventaire des workflows générés :[workflows-index.mdx](../indexes/workflows-index).

## Pipelines principaux

### 1) Qualité + CI

- `test-suite.yml` - vérification des fichiers modifiés + tests dans le navigateur
- `test-v2-pages.yml` - balayage du navigateur v2 + artefacts/comments
- `broken-links.yml` - vérificateur de liens d'alerte

### 2) Pipelines de rafraîchissement des données

- `update-livepeer-release.yml` -> mise à jour des données de version globale
- `update-forum-data.yml` -> mise à jour des extraits de données du forum
- `update-ghost-blog-data.yml` -> mise à jour des extraits de données du blog Ghost
- `update-youtube-data.yml` -> mise à jour des extraits de données de YouTube
- `project-showcase-sync.yml` -> mise à jour des données du pipeline de présentation

### 3) Pipelines de gouvernance + d'entrée

- `discord-issue-intake.yml` + `issue-auto-label.yml`
- workflows d'automatisation de revue (`build-review-assets.yml`, `generate-review-table.yml`, `update-review-template.yml`)

## Notes sur le pipeline de présentation et des sujets tendances

### Présentation

Artéfacts principaux :

- `snippets/automations/showcase/showcaseData.jsx`
- workflow et actifs n8n pour la synchronisation de la présentation

Intention opérationnelle :

- entrées de l'écosystème triables/recherchables
- pipeline réutilisable pour d'autres sources de curation

### Sujets tendances / Agrégation

Artéfacts principaux :

- `snippets/automations/forum/forumData.jsx`
- `snippets/automations/blog/ghostBlogData.jsx`
- `snippets/automations/youtube/youtubeData.jsx`

Intention opérationnelle :

- agrégation des signaux récents de la communauté sur le forum/blog/YouTube
- alimenter les sections « dernières » et maintenir la fraîcheur automatiquement

## Contrôles opérationnels

- Les catalogues générés pour les scripts/workflows/templates évitent les points aveugles.
- Les vérifications CI et pré-validation réduisent le décalage entre les sorties du pipeline et les attentes des documents.
- Lorsque GitHub Actions et n8n se chevauchent, les deux sont conservés pour la flexibilité des mainteneurs.
