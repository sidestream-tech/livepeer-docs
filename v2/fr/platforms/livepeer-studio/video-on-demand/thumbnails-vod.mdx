Pour les actifs VOD, Livepeer Studio peut fournir **des miniatures** pour une utilisation lors du balayage ou des aperçus. Les miniatures sont générées pendant le traitement de l'actif (généralement une image par segment, environ toutes les 3 secondes).

<Info>
  Assets uploaded after November 21, 2023 have thumbnails generated by default. Older assets may need to be re-imported to get thumbnails.
</Info>

## Obtenir les informations de lecture

1. [Télécharger un actif](upload-asset) et attendre qu'il soit prêt (ou utiliser un actif existant).
2. Appelez le [API Info de lecture](https://livepeer.studio/docs/api-reference/playback/get) avec l'**ID de lecture**.

Dans la réponse, cherchez dans `meta.source` pour une entrée avec `hrn: "Thumbnails"` et `type: "text/vtt"`. Cette URL pointe vers un fichier **WebVTT** qui liste les chemins d'images de points clés et les plages de temps.

## format WebVTT

Le fichier VTT liste les segments et les noms de fichiers d'images correspondants, par exemple :

```vtt
WEBVTT
00:00:00.000 --> 00:00:10.000
keyframes_0.jpg
00:00:10.000 --> 00:00:20.000
keyframes_1.jpg
```

## Créer des URL de miniatures

Si l'URL du VTT est :

`https://vod-cdn.lp-playback.studio/.../thumbnails/thumbnails.vtt`

alors la première image clé est généralement :

`https://vod-cdn.lp-playback.studio/.../thumbnails/keyframes_0.jpg`

Utilisez le même chemin de base que l'URL du VTT et ajoutez le nom de fichier du VTT. Vous pouvez choisir l'index du segment qui correspond au moment que vous souhaitez (par exemple, pour un défilement ou une prévisualisation à 30 secondes, utilisez l'image clé appropriée).

## Miniatures pour diffusion en direct

Pour **les diffusions en direct**, les informations de lecture renvoient une seule URL **PNG** (`hrn: "Thumbnail (PNG)"`, `type: "image/png"`) qui met à jour la dernière image. Voir [Lecteur et intégration](../player) pour l'utilisation de l'aperçu/image de couverture dans le Lecteur.
