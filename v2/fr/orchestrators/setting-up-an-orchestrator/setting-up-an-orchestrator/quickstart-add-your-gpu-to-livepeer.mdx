Après [l'installation de go-livepeer](/v2/orchestrators/setup/install-go-livepeer), configurez le nœud afin que les passerelles puissent vous découvrir et envoyer du travail. Cette page couvre les indicateurs principaux et les fichiers de configuration facultatifs.

## Configuration principale

### Réseau et portefeuille

- **-ethUrl** - Arbitrum URL RPC. Voir [Connectez-vous à Arbitrum](/v2/orchestrators/setup/install-go-livepeer).
- **-ethAcctAddr** - (Facultatif) adresse du compte ETH si vous avez déjà un keystore.
- **-network** - Utilisez `arbitrum-one-mainnet` pour la production.

### Identité de l'Orchestrator et tarification

- **-serviceAddr** - Hôte public ou IP et port (par exemple `orchestrator.example.com:8935`). Privilégiez un hôte que vous contrôlez ; modifier cet hôte plus tard nécessite une mise à jour sur la chaîne.
- **-pricePerUnit** - Prix en wei par pixel (transcodage). Requis au démarrage ; peut être mis à jour plus tard via la CLI ou sur la chaîne.
- **-pixelsPerUnit** - Pixels par unité de tarification (par défaut 1). Affecte la manière dont le prix est calculé par segment.

### GPU et mode

- **-orchestrator** - Exécuter en mode orchestrateur.
- **-transcodeur** - Activer la transcodification GPU.
- **-nvidia** - Identifiants NVIDIA GPU séparés par des virgules (par exemple "`0,1`). Omettre pour une utilisation uniquement CPU.

Exemple d'exécution minimale :

<CustomCodeBlock
  codeString={`livepeer \\
  -orchestrator \\
  -transcoder \\
  -network arbitrum-one-mainnet \\
  -ethUrl <ETH_URL> \\
  -serviceAddr <HOST:8935> \\
  -pricePerUnit <WEI_PER_PIXEL> \\
  -nvidia 0,1`}
  language="bash"
  icon="terminal"
/>

Vous pouvez définir des profils de transcodage dans un fichier JSON et les référencer. Structure d'exemple :

## Optionnel : modèles d'IA (orchestrateurs d'IA)

Si vous exécutez une inférence d'IA, créez un**aiModels.json** pour spécifier les pipelines et les modèles :

<CustomCodeBlock
  codeString={`[
  {
    "pipeline": "text-to-image",
    "model_id": "stabilityai/sd-turbo",
    "warm": true
  }
]`}
  language="json"
  icon="brackets-curly"
  filename="aiModels.json"
/>

Voir [Pipelines IA](../../advanced-setup/ai-pipelines) et [Développeurs BYOC](/developers/ai-pipelines/byoc) pour une configuration complète d'IA.

## Vérification

Après le démarrage du nœud :

- Les journaux affichent`Listening for RPC on :8935` et aucun erreur RPC.
- [Explorateur](https://explorer.livepeer.org/) affiche votre nœud après l'activation.
- Sous charge, l'utilisation de la GPU (par exemple, `nvidia-smi`) augmente lorsque les tâches sont traitées.
