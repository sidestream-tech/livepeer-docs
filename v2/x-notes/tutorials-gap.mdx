Short answer: **not very realistic** unless they are written to run against hosted infrastructure.

There are three distinct developer realities:

---

# 1️⃣ Developer Types

## A. App Developer (does NOT run infra)

They:

* Do not run a gateway
* Do not run an orchestrator
* Do not manage GPUs
* Just want: ingest → AI → playback via API

For them:

* Tutorials requiring a local gateway/orchestrator are **non-starters**
* BYOC infra guides are irrelevant
* Smart contract knowledge is unnecessary

If tutorials assume infra control, adoption collapses.

---

## B. AI Builder (wants pipeline control, not network ops)

They:

* Want to modify Comfy workflows
* Want to run models
* Want to test latency
* Do not want to run staking nodes

For them:

* Local `ai-runner` + ComfyStream is realistic
* Full orchestrator setup is not

---

## C. Network Operator (Orchestrator / Gateway)

They:

* Do run infra
* Do manage GPUs
* Care about performance + economics
* Need full stack guides

---

# 2️⃣ The Core Problem

Most current technical materials skew toward:

* Operator setup
* Protocol internals
* GPU config
* Infrastructure

But **most potential developers are Type A or B**.

If a tutorial requires:

* Running go-livepeer
* Configuring staking
* Connecting ai-worker manually
* Managing node keys

…it is effectively limited to a tiny audience.

---

# 3️⃣ What Is Actually Realistic

Realistic tutorials must support:

### Option 1 - Hosted Path

Use:

* Studio API
* Daydream API
* Managed AI endpoint

No node required.

---

### Option 2 - Local AI Only

Use:

* ComfyUI
* ai-runner
* Local GPU

No orchestrator required.

---

### Option 3 - Hybrid (Advanced)

Use:

* Local ai-runner
* Remote gateway
* No staking required

---

# 4️⃣ The Honest Answer

If tutorials require:

* Gateway + orchestrator + staking

They are realistic only for:

* Existing network operators
* Web3-native infra teams

Not for mainstream AI devs.

---

# 5️⃣ The Structural Insight

Livepeer’s supply side (nodes, staking) and demand side (AI builders) are separate markets.

Tutorials must reflect that separation.

Right now:

* Supply-side tutorials exist.
* Demand-side tutorials are fragmented.

---

# 6️⃣ What Should Happen Instead

A realistic tutorial ladder would look like:

1. 10-minute “Realtime AI Filter” (zero infra)
2. Custom pipeline tutorial (local GPU only)
3. Deploy pipeline to remote AI endpoint
4. Scale to multi-stream
5. Then - optional - run your own worker
6. Then - optional - full orchestrator

Infrastructure should be the final step, not the first.
