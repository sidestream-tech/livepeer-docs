Use this page to confirm the node is actually live after setup, activation, and first workload tests.

## Planned checks

<AccordionGroup>
  <Accordion title="Reachability" icon="plug">
    **Symptom**
    The node starts, but you are not sure external systems can reach it.

    **Cause**
    A correct local startup does not guarantee that `serviceAddr`, RPC, or public reachability are correct.

    **Fix**
    Verify `serviceAddr`, activation details, and the public-facing node path.

    **Verify**
    Use the existing setup test, Explorer checks, and the activation page together.
  </Accordion>
  <Accordion title="First job" icon="film">
    **Symptom**
    The node is up but has not shown a clear workload success signal yet.

    **Cause**
    Activation, pricing, or routing may still be incomplete.

    **Fix**
    Run the default quickstart validation path and compare it with active-set and routing checks.

    **Verify**
    Confirm the node accepts a first workload and shows the expected logs and health signals.
  </Accordion>
  <Accordion title="First earnings" icon="coins">
    **Symptom**
    The node is active but you have no visible payout, fee, or rewards signal yet.

    **Cause**
    Rewards, fees, and first observable earnings often appear at different checkpoints.

    **Fix**
    Review activation, Explorer visibility, and the current earnings guidance.

    **Verify**
    Confirm the first visible on-chain or Explorer signal you expect for your chosen path.
  </Accordion>
</AccordionGroup>
