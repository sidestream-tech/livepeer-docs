Orchestrators on the Livepeer Network perform compute work for the network, which can be broadly categorized into three main types:
- GPU nodes that perform [video transcoding](/v2/developers/build/video-transcoding)
- AI nodes that perform [AI work](/guides/ai-pipelines)
- Custom Workloads ie. [BYOC pipelines](/v2/developers/build/byoc) or application-specific workloads (e.g. [Embody](/v2/solutions/embody) AI avatars or [Daydream](/v2/solutions/daydream) real-time AI video workloads)


## General Compute Functions

Orchestrators perform two core compute functions on the Livepeer network: transcoding and AI inference.
They do this by running a node that is connected to the network and performing the work that is assigned to it.

#### Transcoding

Transcoding is the process of converting video from one format to another.
Orchestrators on the Livepeer network perform transcoding by running a node that is connected to the network and performing the work that is assigned to it.

#### AI Inference

AI inference is the process of running a model to make a prediction. Orchestrators on the Livepeer network perform AI inference by running a node that is connected to the network and performing the work that is assigned to it.

#### Real-Time AI Video Pipelines
Real-time AI video pipelines are a new type of workload that orchestrators can run on the Livepeer network. These pipelines allow for real-time processing of video using AI models, enabling applications like interactive live video, real-time avatars, and more.

#### Custom Workloads
Orchestrators can also run custom workloads on the Livepeer network - both CPU and GPU workloads via BYOC workflows. These containerise workloads allow orchestrators to run workloads defined by applications or users, as well as application-specific workloads like the Embody AI avatars or Daydream real-time AI video workloads.
