**go-livepeer** needs access to the [Arbitrum](https://arbitrum.io/) network. You can use a **hosted API** (e.g. [Infura](https://infura.io/), [Alchemy](https://www.alchemy.com/)) or a **self-hosted** node.

## Hosted RPC (recommended to start)

<CustomCodeBlock
  codeString={`livepeer -network arbitrum-one-mainnet -ethUrl https://arbitrum-mainnet.infura.io/v3/<PROJECT_ID>`}
  language="bash"
  icon="terminal"
/>

Replace `<PROJECT_ID>` with your Infura project ID.

## Self-hosted node

See [Offchain Labs docs](https://developer.offchainlabs.com/docs/running_node). Then:

<CustomCodeBlock
  codeString={`livepeer -network mainnet -ethUrl http://localhost:8545`}
  language="bash"
  icon="terminal"
/>

## Supported networks

- **Arbitrum One** (production): `-network arbitrum-one-mainnet -ethUrl <RPC_URL>`
- **Private / offchain:** See [CLI flags](/v2/orchestrators/resources/cli-flags) for `-network` and `-ethController`.

<Note> Use a stable RPC; flaky endpoints can cause missed reward calls.</Note>
