You can install **go-livepeer** using a binary release, Docker, or by building from source. Docker is the most common choice for production.

## Prerequisites

- **Dependencies (binary):** `curl`, `coreutils`, `gnupg2` (Linux) or `gnupg` (macOS).
- **Docker (Docker install):** [Install Docker](https://docs.docker.com/get-docker/) and, for GPU support, [NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html).
- **GPU (transcoding):** NVIDIA driver; for Linux GPU binary or build from source, [CUDA Toolkit](https://developer.nvidia.com/cuda-downloads) (e.g. v12).

## Binary release

Replace `<RELEASE_VERSION>` with a release tag (e.g. `v0.5.35`) and `<ARCH>` with `amd64` or `arm64`.

### Darwin (macOS)

<CustomCodeBlock
  codeString={`wget https://github.com/livepeer/go-livepeer/releases/download/<RELEASE_VERSION>/livepeer-darwin-<ARCH>.tar.gz
tar -zxvf livepeer-darwin-<ARCH>.tar.gz
mv livepeer-darwin-<ARCH>/* /usr/local/bin/`}
  language="bash"
  icon="terminal"
/>

### Linux

<CustomCodeBlock
  codeString={`wget https://github.com/livepeer/go-livepeer/releases/download/<RELEASE_VERSION>/livepeer-linux-<ARCH>.tar.gz
tar -zxvf livepeer-linux-<ARCH>.tar.gz
mv livepeer-linux-<ARCH>/* /usr/local/bin/`}
  language="bash"
  icon="terminal"
/>

### Linux (GPU / NVIDIA)

Use the GPU build if you will run transcoding on NVIDIA GPUs. Requires CUDA 12.

<CustomCodeBlock
  codeString={`wget https://github.com/livepeer/go-livepeer/releases/download/<RELEASE_VERSION>/livepeer-linux-gpu-<ARCH>.tar.gz
tar -zxvf livepeer-linux-gpu-<ARCH>.tar.gz
mv livepeer-linux-gpu-<ARCH>/* /usr/local/bin/`}
  language="bash"
  icon="terminal"
/>

### Windows

<CustomCodeBlock
  codeString={`wget https://github.com/livepeer/go-livepeer/releases/download/<RELEASE_VERSION>/livepeer-windows-amd64.zip
unzip livepeer-windows-amd64.zip
move livepeer-windows-amd64 C:\\Users\\UserName\\livepeer-folder`}
  language="shell"
  icon="terminal"
/>

<Info> Livepeer does not provide automatic updates. You can update manually or use a community script (e.g. [Forum: Bash script to update Livepeer](https://forum.livepeer.org/t/bash-script-to-update-livepeer/1513)).</Info>

## Docker

Images are published to [Docker Hub](https://hub.docker.com/r/livepeer/go-livepeer).

<CustomCodeBlock
  codeString={`docker pull livepeer/go-livepeer:<RELEASE_VERSION>`}
  language="bash"
  icon="docker"
/>

For the latest pre-release:

<CustomCodeBlock
  codeString={`docker pull livepeer/go-livepeer:master`}
  language="bash"
  icon="docker"
/>

Run with your config flags; for GPU support use `--gpus all`:

<CustomCodeBlock
  codeString={`docker run --gpus all <image_id> -orchestrator -transcoder -network arbitrum-one-mainnet -ethUrl <ETH_URL> ...`}
  language="bash"
  icon="docker"
/>

To use the CLI from the host:

<CustomCodeBlock
  codeString={`docker exec -it <container_name> livepeer_cli`}
  language="bash"
  icon="terminal"
/>

## Build from source

1. **System deps (Ubuntu 20.04+):**

<CustomCodeBlock
  codeString={`apt-get -y install protobuf-compiler-grpc golang-goprotobuf-dev yasm pkg-config`}
  language="bash"
  icon="terminal"
/>

2. **Go:** Install [Go](https://go.dev/doc/install).

3. **Clone and build:**

<CustomCodeBlock
  codeString={`git clone https://github.com/livepeer/go-livepeer.git
cd go-livepeer
./install_ffmpeg.sh
export PKG_CONFIG_PATH=~/compiled/lib/pkgconfig
export BUILD_TAGS=mainnet
make
cp livepeer* /usr/local/bin`}
  language="bash"
  icon="terminal"
/>

<Note> For GPU support on Linux, install the [CUDA Toolkit](https://developer.nvidia.com/cuda-toolkit) and set `LIBRARY_PATH`; install `clang` if required by the ffmpeg script.</Note>

## Activate

After installation, [run the node in orchestrator mode](../quickstart/orchestrator-setup), fund your account with ETH and LPT, then activate:

1. Run **livepeer_cli** and choose the multi-step “become an orchestrator” option.
2. Set **block reward cut** and **fee share** (percent kept vs shared with delegators).
3. Set **pixels per unit** and **price per unit** (wei).
4. Set **service address** (must match `-serviceAddr` used when starting the node).
5. **Bond** LPT (amount in LPTU, e.g. 5000000000000000000 for 5 LPT).
6. Wait for transactions to confirm; your orchestrator joins the active set in the next round.

See [Staking LPT](../advanced-setup/staking-LPT) and [Rewards and fees](../advanced-setup/rewards-and-fees) for economics. Check the [Explorer](https://explorer.livepeer.org/leaderboard) to confirm your node and the current minimum stake for the top 100.
