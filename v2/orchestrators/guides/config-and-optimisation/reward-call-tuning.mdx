Every round (approximately 22 hours on Arbitrum), an active orchestrator calls `Reward()` on-chain to claim its proportional share of newly minted LPT. The call is a gas transaction on Arbitrum One. For orchestrators with small stake, the gas cost of the call can exceed the LPT value received — at which point calling reward destroys value rather than creating it.

This page covers: the profitability calculation, the `-reward` flag for enabling or disabling automatic calling, manual calling via `livepeer_cli`, timing considerations, and when to skip rounds intentionally.

<CustomDivider middleText="Profitability Threshold" />

## Profitability calculation

Reward calling is profitable when the LPT received (expressed in ETH) exceeds the gas cost of the transaction.

**Gas cost per reward call:**

```text icon="terminal"
gas cost (ETH) = gas_used × gas_price_in_ETH

Typical gas used:       350,000 – 450,000 gas
Arbitrum gas price:     0.01 – 0.1 gwei (1 gwei = 1e-9 ETH)
Approximate ETH cost:   0.0000035 – 0.000045 ETH
```

**LPT value per round:**

```text icon="terminal"
LPT earned = (your total stake / total active stake) × round_issuance × 0.90

0.90 = 90% remaining after the 10% LIP-89 Treasury allocation
```

Round issuance varies with the current inflation rate. The Explorer shows current inflation at [explorer.livepeer.org/orchestrators](https://explorer.livepeer.org/orchestrators).

**Break-even stake (illustrative):**

At a typical Arbitrum gas cost of ~0.00002 ETH and LPT price of ~$4 USD, the minimum stake for profitable calling is approximately several thousand LPT. The exact threshold moves with ETH/LPT price ratio and gas conditions.

## Automatic vs manual calling

### Automatic calling (default)

By default, go-livepeer calls `Reward()` automatically at the start of each new round. No action required after initial activation.

```bash icon="terminal"
# Automatic reward is ON by default.
# Confirm no -reward=false in your startup flags.
livepeer \
    -network arbitrum-one-mainnet \
    -orchestrator \
    ...
```

Requirements for successful automatic calling:
- Node must be running at round initialisation (~22-hour intervals)
- Orchestrator wallet must hold sufficient ETH on Arbitrum for the transaction
- Orchestrator must be in the active set that round

<Warning>
  If your node is offline at round initialisation, that round's reward is permanently missed. Use a systemd service or Docker restart policy (`restart: unless-stopped`) to keep the node running continuously.
</Warning>

### Manual calling

Disable automatic reward calls with `-reward=false` and call manually to evaluate profitability before committing:

```bash icon="terminal"
livepeer \
    -network arbitrum-one-mainnet \
    -reward=false \
    ...
```

Manual call via `livepeer_cli`:

1. Verify the node is running
2. Run `livepeer_cli`
3. Select the `Invoke "reward"` option
4. Wait for the transaction to confirm — confirmation appears in node logs

```bash icon="terminal"
# Monitor logs for reward confirmation
journalctl -u livepeer -f | grep -i "reward"
```

## Timing considerations

Arbitrum gas prices fluctuate. Calling early in a round (within the first minutes of initialisation) guarantees the call succeeds for that round but may coincide with elevated gas if multiple orchestrators call simultaneously. Calling later in the round carries the risk of node downtime before the deadline.

**For most operators:** leave automatic calling enabled. The gas savings from optimal timing are negligible on Arbitrum compared to the risk of a missed round from manual management.

**For low-stake operators evaluating profitability:** run with `-reward=false` for 3 to 5 rounds. Check Arbiscan for current gas costs and use the Explorer to read the LPT value for those rounds. If the gas cost exceeds the LPT value at current prices, continue without calling and accumulate stake from delegation before enabling automatic calling.

### When to skip calling intentionally

Skip calling only when the profitability calculation is definitively negative. At Arbitrum gas prices, the break-even point is much lower than it was historically on Ethereum L1. Most operators with more than a few thousand LPT delegated will find automatic calling profitable.

Accumulating without calling does not compound — unclaimed LPT does not accrue additional inflation. The LPT you would have received each round is lost, not deferred.

## Monitoring reward calls

### Alert for missed rounds

Set a Prometheus alert on the `livepeer_round_last_claim` metric. A value that does not advance each round indicates a missed call.

{/* REVIEW: Confirm exact Prometheus metric name for last successful reward call / round tracking. Rick to verify against current go-livepeer metrics export. */}

### Check call history

Your orchestrator's reward call history is visible on [Livepeer Explorer](https://explorer.livepeer.org/orchestrators) on your orchestrator page. Each round shows whether the call succeeded, was skipped, or was missed.

Monitor your ETH balance on Arbitrum via [Arbiscan](https://arbiscan.io/). The reward call gas cost appears as an outbound ETH transaction approximately every 22 hours when automatic calling is active.
