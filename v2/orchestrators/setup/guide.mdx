<Tip>
  Setup covers the technical readiness of the node: installed, configured, staked, and verified. Earning - pricing, delegation strategy, and workload optimisation - is in the Guides section.
</Tip>

This section takes an operator from "decided to run a node" to "node is on-chain, processing jobs, and verified." Each page is one session of work.

<Note>
  If still evaluating whether to operate, see <a href="/v2/orchestrators/guides/operator-considerations/operator-rationale">Operator Rationale</a> first. If the hardware is not yet verified, start with the <a href="/v2/orchestrators/quickstart/guide">Quickstart</a>.
</Note>

## Setup flow

<StyledSteps>

  <StyledStep title="Install go-livepeer" icon="download">
    Install the go-livepeer binary. Docker is recommended - it handles GPU passthrough and matches the quickstart environment.

    <a href="/v2/orchestrators/setup/rs-install">→ Install go-livepeer</a>
  </StyledStep>

  <StyledStep title="Configure" icon="sliders">
    Set operational flags for the chosen node mode: video transcoding, AI inference, or dual mode. Generates a docker-compose file with all required flags.

    <a href="/v2/orchestrators/setup/configure">→ Configure your orchestrator</a>
  </StyledStep>

  <StyledStep title="Connect to Arbitrum" icon="link">
    Connect to Arbitrum One: fund the wallet with arbETH, acquire and stake LPT, register the service URI, and activate on-chain. This is the longest step - LPT acquisition can take hours to days.

    <a href="/v2/orchestrators/setup/connect-and-activate">→ Connect and activate</a>
  </StyledStep>

  <StyledStep title="Verify" icon="circle-check">
    Confirm the node is on-chain, transcoding video, running AI inference (if configured), calling reward correctly, and exposing metrics.

    <a href="/v2/orchestrators/setup/test">→ Verify your setup</a>
  </StyledStep>

</StyledSteps>

## What you will need

<StyledTable variant="bordered">
  <thead>
    <TableRow header>
      <TableCell header>Category</TableCell>
      <TableCell header>Requirement</TableCell>
      <TableCell header>Notes</TableCell>
    </TableRow>
  </thead>
  <tbody>
    <TableRow>
      <TableCell>**Hardware**</TableCell>
      <TableCell>NVIDIA GPU</TableCell>
      <TableCell>Any model for video transcoding. 24 GB VRAM for AI diffusion pipelines; 8 GB for LLM via Ollama. See the Requirements guide for per-pipeline VRAM floors.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>**Software**</TableCell>
      <TableCell>Docker + NVIDIA Container Toolkit</TableCell>
      <TableCell>Recommended path. Binary install also supported for Linux, Windows, macOS.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>**Blockchain**</TableCell>
      <TableCell>arbETH + LPT on Arbitrum One</TableCell>
      <TableCell>~0.05 ETH for gas. LPT stake: check current active set threshold on Explorer. AI-only nodes need minimal LPT.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>**Network**</TableCell>
      <TableCell>Public IP or domain; port 8935 open</TableCell>
      <TableCell>Gateways connect on port 8935. Static IP or DNS strongly recommended.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>**RPC**</TableCell>
      <TableCell>Arbitrum One RPC endpoint</TableCell>
      <TableCell>Free tier from Alchemy or Infura is sufficient for most operators.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>**Time**</TableCell>
      <TableCell>2-4 hours for video; 4-8 hours for AI or dual mode</TableCell>
      <TableCell>Most of the time is LPT acquisition and model download, not active configuration.</TableCell>
    </TableRow>
  </tbody>
</StyledTable>

For the full hardware and software requirements checklist, see <a href="/v2/orchestrators/guides/operator-considerations/requirements">Guides > Operator Considerations > Requirements</a>.
