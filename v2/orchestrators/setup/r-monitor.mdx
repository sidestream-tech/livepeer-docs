This page covers the minimum monitoring you should have in place once your orchestrator is live. For full observability (Grafana dashboards, alerting, DCGM exporter), see the [Metrics & Monitoring guide](/v2/orchestrators/guides/monitoring-and-tooling/metrics-and-alerting).

<CustomDivider />

## Enable Prometheus metrics

Add `-monitor` to your go-livepeer startup command:

```bash icon="terminal"
livepeer \
  -network arbitrum-one-mainnet \
  -ethUrl <YOUR_RPC_URL> \
  -orchestrator \
  -transcoder \
  -nvidia 0 \
  -monitor \
  ...
```

Metrics are exposed at:

``` icon="terminal"
http://localhost:7935/metrics
```

Verify with:

```bash icon="terminal"
curl -s http://localhost:7935/metrics | head -20
```

## Check Explorer

After each round (~24 hours), verify on [Livepeer Explorer](https://explorer.livepeer.org):

1. **Status** — should show **Active**
2. **Reward calls** — should show a reward for each round your node was online
3. **Fees earned** — will appear after you process transcoding or AI jobs

If reward calls are missing, check:
- Your node was online and connected during the round
- You did **not** start with `-reward=false`
- Your Arbitrum ETH balance is sufficient for gas

## GPU monitoring

Check GPU health and utilisation:

```bash icon="terminal"
# Current status
nvidia-smi

# Continuous monitoring (updates every 2 seconds)
nvidia-smi -l 2

# CSV output for logging: utilisation, memory, temperature
nvidia-smi --query-gpu=utilization.gpu,utilization.memory,memory.used,memory.total,temperature.gpu --format=csv -l 5
```

Watch for:
- **Temperature** above 85°C — indicates inadequate cooling
- **Memory utilisation** near 100% — may cause out-of-memory errors for AI workloads
- **GPU utilisation** at 0% when you should be processing jobs — check go-livepeer logs

## Minimal Prometheus setup

If you want to scrape and store metrics, add a `prometheus.yml`:

```yaml icon="terminal"
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'go-livepeer'
    static_configs:
      - targets: ['localhost:7935']
```

Run Prometheus with Docker:

```bash icon="terminal"
docker run -d \
  --name prometheus \
  --network host \
  -v $(pwd)/prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus
```

Access the Prometheus UI at `http://localhost:9090`.
