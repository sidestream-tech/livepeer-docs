This page covers the minimum monitoring you should have in place once your orchestrator is live. For full observability (Grafana dashboards, alerting, DCGM exporter), see the [Metrics & Monitoring guide](/v2/orchestrators/guides/monitoring-and-troubleshooting/metrics-monitoring).

---

## Enable Prometheus metrics

Add `-monitor` to your go-livepeer startup command:

```bash
livepeer \
  -network arbitrum-one-mainnet \
  -ethUrl <YOUR_RPC_URL> \
  -orchestrator \
  -transcoder \
  -nvidia 0 \
  -monitor \
  ...
```

Metrics are exposed at:

```
http://localhost:7935/metrics
```

Verify with:

```bash
curl -s http://localhost:7935/metrics | head -20
```

---

## Key metrics to watch

| Metric | What it tells you |
|---|---|
| `segment_source_appeared_total` | Total segments received — confirms you are getting jobs |
| `segment_transcoded_total` | Successfully transcoded segments |
| `segment_transcode_errors_total` | Failed transcodes — high values reduce gateway selection probability |
| `transcode_latency_seconds` | Time to transcode each segment — should stay well under real-time |
| `current_round` | Current Livepeer protocol round |
| `reward_call_count` | Number of reward calls submitted — should increment once per round |

For AI inference metrics, also watch `ai_request_count` and `ai_request_latency_seconds`.

## Check Explorer

After each round (~24 hours), verify on [Livepeer Explorer](https://explorer.livepeer.org):

1. **Status** — should show **Active**
2. **Reward calls** — should show a reward for each round your node was online
3. **Fees earned** — will appear after you process transcoding or AI jobs

If reward calls are missing, check:
- Your node was online and connected during the round
- You did **not** start with `-reward=false`
- Your Arbitrum ETH balance is sufficient for gas

## GPU monitoring

Check GPU health and utilisation:

```bash
# Current status
nvidia-smi

# Continuous monitoring (updates every 2 seconds)
nvidia-smi -l 2

# CSV output for logging: utilisation, memory, temperature
nvidia-smi --query-gpu=utilization.gpu,utilization.memory,memory.used,memory.total,temperature.gpu --format=csv -l 5
```

Watch for:
- **Temperature** above 85°C — indicates inadequate cooling
- **Memory utilisation** near 100% — may cause out-of-memory errors for AI workloads
- **GPU utilisation** at 0% when you should be processing jobs — check go-livepeer logs

## Quick symptom guide

| Symptom | Likely cause | Action |
|---|---|---|
| No segments in logs | Service address unreachable or price too high | Re-check port 8935 from external machine; compare pricing on Explorer |
| Reward calls missing | Node offline during round, or `-reward=false` | Confirm uptime; check startup flags |
| High transcode errors | GPU driver issue or CUDA mismatch | Run `nvidia-smi`; check go-livepeer error logs |
| RPC errors in logs | Arbitrum endpoint down or rate-limited | Test RPC with `curl`; switch to a different provider if needed |
| AI runner not responding | Container crashed or port conflict | `docker logs <container>`; check `curl localhost:8000/health` |

For detailed troubleshooting, see the [FAQ](/v2/orchestrators/resources/faq).

## Minimal Prometheus setup

If you want to scrape and store metrics, add a `prometheus.yml`:

```yaml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: 'go-livepeer'
    static_configs:
      - targets: ['localhost:7935']
```

Run Prometheus with Docker:

```bash
docker run -d \
  --name prometheus \
  --network host \
  -v $(pwd)/prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus
```

Access the Prometheus UI at `http://localhost:9090`.
