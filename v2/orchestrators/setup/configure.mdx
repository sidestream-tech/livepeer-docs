## Video transcoding configuration

Video transcoding is the baseline mode. The orchestrator accepts video segments from gateways, transcodes them on the GPU via NVENC/NVDEC, and returns HLS output.

**docker-compose template:**

```yaml icon="code" filename="docker-compose.yml"
services:
  orchestrator:
    image: livepeer/go-livepeer:stable
    container_name: livepeer-orchestrator
    restart: unless-stopped
    network_mode: host
    volumes:
      - ~/.lpData:/root/.lpData
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
    command:
      - -network=arbitrum-one-mainnet
      - -ethUrl=https://arb-mainnet.g.alchemy.com/v2/YOUR_API_KEY
      - -orchestrator
      - -transcoder
      - -nvidia=0
      - -maxSessions=10
      - -pricePerUnit=1000
      - -serviceAddr=YOUR_PUBLIC_IP:8935
      - -monitor
      - -v=6
```

Replace:
- `YOUR_API_KEY` with your Alchemy or Infura Arbitrum One API key
- `YOUR_PUBLIC_IP` with the machine's public IP address or domain name
- `-nvidia=0` with the GPU device ID(s) from `nvidia-smi -L`
- `-maxSessions=10` with your calculated session limit (see <LinkArrow href="/v2/orchestrators/guides/config-and-optimisation/capacity-planning" label="Capacity Planning" newline={false} />)
- `-pricePerUnit=1000` with a competitive price in wei per pixel

## AI inference configuration

AI inference serves inference requests (text-to-image, audio-to-text, LLM, and other pipelines) from gateways and applications. AI routing is capability-based - no top-100 stake required to receive AI jobs.

**docker-compose template:**

```yaml icon="code" filename="docker-compose.yml"
services:
  orchestrator:
    image: livepeer/go-livepeer:stable
    container_name: livepeer-ai-orchestrator
    restart: unless-stopped
    network_mode: host
    volumes:
      - ~/.lpData:/root/.lpData
      - /var/run/docker.sock:/var/run/docker.sock
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
    command:
      - -network=arbitrum-one-mainnet
      - -ethUrl=https://arb-mainnet.g.alchemy.com/v2/YOUR_API_KEY
      - -orchestrator
      - -transcoder
      - -nvidia=0
      - -pricePerUnit=1000
      - -serviceAddr=YOUR_PUBLIC_IP:8935
      - -aiWorker
      - -aiModels=/root/.lpData/aiModels.json
      - -aiModelsDir=/root/.lpData/models
      - -monitor
      - -v=6
```

**Required: create `~/.lpData/aiModels.json`** before starting:

```json icon="code" filename="~/.lpData/aiModels.json"
[
  {
    "pipeline": "text-to-image",
    "model_id": "ByteDance/SDXL-Lightning",
    "price_per_unit": 4768371,
    "warm": true
  }
]
```

Check current demand and pricing at [tools.livepeer.cloud/ai/network-capabilities](https://tools.livepeer.cloud/ai/network-capabilities) before setting `price_per_unit`. Gateways skip orchestrators priced above their cap.

**First start downloads model weights** from HuggingFace. A 6 GB model like SDXL-Lightning takes 5-30 minutes depending on connection speed. The AI runner starts serving requests after the download completes.

**For the LLM pipeline (8-16 GB VRAM):** the `llm` pipeline uses the Ollama runner instead of `livepeer/ai-runner`. See <LinkArrow href="/v2/orchestrators/guides/ai-and-job-workloads/llm-pipeline-setup" label="LLM Pipeline Setup" newline={false} /> for the full Ollama docker-compose setup.

## Dual mode configuration (video + AI)

Dual mode serves both video transcoding and AI inference from one go-livepeer process. Video uses NVENC/NVDEC dedicated hardware. AI uses CUDA compute cores. VRAM is the shared capacity limit.

**docker-compose template:**

```yaml icon="code" filename="docker-compose.yml"
services:
  orchestrator:
    image: livepeer/go-livepeer:stable
    container_name: livepeer-dual-orchestrator
    restart: unless-stopped
    network_mode: host
    volumes:
      - ~/.lpData:/root/.lpData
      - /var/run/docker.sock:/var/run/docker.sock
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
    command:
      - -network=arbitrum-one-mainnet
      - -ethUrl=https://arb-mainnet.g.alchemy.com/v2/YOUR_API_KEY
      - -orchestrator
      - -transcoder
      - -nvidia=0
      - -maxSessions=10
      - -pricePerUnit=1000
      - -serviceAddr=YOUR_PUBLIC_IP:8935
      - -aiWorker
      - -aiModels=/root/.lpData/aiModels.json
      - -aiModelsDir=/root/.lpData/models
      - -monitor
      - -v=6
```

**Create `~/.lpData/aiModels.json`** with at least one pipeline entry before starting. Same format as the AI tab above.

**VRAM planning:** NVENC/NVDEC use dedicated silicon and consume negligible VRAM. AI model weights set the practical limit. A 24 GB GPU leaves room for video sessions alongside one warm SDXL-Lightning model (~12-18 GB).

## Common configuration

### Networking

The `-serviceAddr` flag declares the public address gateways connect to. This address must be reachable from the internet:

```bash icon="terminal" filename="serviceAddr-examples"
# IP address
-serviceAddr 203.0.113.42:8935

# Domain name (preferred - survives IP changes without re-registration)
-serviceAddr orch.yourdomain.com:8935
```

Port 8935 must be open inbound on the firewall. Test reachability **from a different machine** before connecting on-chain:

```bash icon="terminal" filename="port-check"
curl -k https://YOUR_PUBLIC_IP:8935/status
```

Any response (including a JSON error) confirms the port is open. A connection timeout means the firewall is blocking it.

Port 7935 (`-cliAddr`) is for local CLI access and Prometheus metrics. Keep it bound to localhost unless external monitoring is configured.

### GPU selection

```bash icon="terminal" filename="nvidia-list"
nvidia-smi -L
```

This lists GPU device IDs:
```text icon="terminal" filename="expected-gpu-list"
GPU 0: NVIDIA GeForce RTX 4090 (UUID: GPU-...)
GPU 1: NVIDIA GeForce RTX 3090 (UUID: GPU-...)
```

Use the device index in `-nvidia`:
```bash icon="terminal" filename="nvidia-examples"
-nvidia 0        # Single GPU
-nvidia 0,1      # Two GPUs
-nvidia all      # All available GPUs
```

For mixed GPU configurations (different VRAM sizes), assign specific GPU IDs to specific pipelines in `aiModels.json` using the `gpu` field.

### Pricing

`-pricePerUnit` sets video transcoding price in **wei per pixel** (not ETH). A typical starting range is 500-2,000 wei per pixel. Set it below 1,000 wei per pixel initially and adjust based on job volume.

For AI pricing, set `price_per_unit` per pipeline in `aiModels.json`. Values are in wei per output pixel (or per ms for audio, per token for LLM).

For competitive positioning guidance, see <LinkArrow href="/v2/orchestrators/guides/config-and-optimisation/pricing-strategy" label="Pricing Strategy" newline={false} />.

### Persistent data

Always mount the data directory so the keystore and data survive container restarts:

```yaml icon="code" filename="volume-mount"
volumes:
  - ~/.lpData:/root/.lpData
```

Without this mount, go-livepeer creates a new Ethereum account on every container start, losing the previous keystore and all bonded LPT.

For AI workloads, also mount the models directory:

```yaml icon="code" filename="models-mount"
volumes:
  - ~/.lpData:/root/.lpData
  - /var/run/docker.sock:/var/run/docker.sock
```

The `aiModelsDir` must be a **host path**. Docker mounts that path into AI runner containers.

## Verify configuration before connecting on-chain

Start the node and confirm it initialises cleanly before proceeding to the Connect step:

```bash icon="terminal" filename="start"
docker compose up -d
```

Check for clean startup:

```bash icon="terminal" filename="check-logs"
docker compose logs -f 2>&1 | grep -iE "gpu|transcode|ai-runner|error|FATAL" | head -20
```

Expected for video mode:
```text icon="terminal" filename="expected-video-startup"
Transcoding on Nvidia GPU 0
Listening for RPC on :8935
```

Expected for AI mode (after model download):
```text icon="terminal" filename="expected-ai-startup"
Starting AI worker
Warm model loaded: ByteDance/SDXL-Lightning
```

Resolve `FATAL` or repeated error lines before connecting on-chain.
