This quickstart is the default path for solo operators who want to get a video-first go-livepeer orchestrator running as quickly as possible.

## What this quickstart does

- Gets a solo operator from install to a first working node.
- Focuses on the default go-livepeer path.
- Optimizes for speed to first working setup, not deep customization.

## What this quickstart does not do

- It does not cover joining a pool.
- It does not cover non-default split or custom setups.
- It does not serve as the main AI workload onboarding path.
- It does not cover fleet or enterprise operating models.

## Default quickstart scope

The default quickstart is video-first.

- Use [Video Transcoding Quickstart](/v2/orchestrators/quickstart/video-transcoding) for the main step-by-step path.
- Use the setup guide if you need more control over install, configuration, connect, publish, and verification.
- Use Advanced Operations for AI specialization, fleet concerns, or operational tuning.

## First Job

Your first success checkpoint is a working transcoding job.

- Run the detailed path in [Video Transcoding Quickstart](/v2/orchestrators/quickstart/video-transcoding).
- Complete network activation in [Activate on the Network](/v2/orchestrators/unclassified/rcs-connect-activate-publish).
- Verify the node responds and can accept work in [Testing Your Orchestrator Setup](/v2/orchestrators/setup/x-test).

## First Earnings

Your next checkpoint is seeing rewards or fee activity show up where you can verify it.

- Review [Orchestrator Earnings Explained](/v2/orchestrators/operations/earnings).
- Review [Rewards and Fees](/v2/orchestrators/operations/rs-rewards-and-fees).
- Use [FAQ and Troubleshooting](/v2/orchestrators/resources/faq) if the node is active but not receiving work.
