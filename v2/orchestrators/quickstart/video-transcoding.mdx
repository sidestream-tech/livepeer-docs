This quickstart gets a transcoding orchestrator running with Docker using opinionated defaults. By the end, your node will be registered on the Livepeer network and ready to receive video transcoding jobs.

<Note>
This guide uses Docker with GPU passthrough. For binary installs, macOS/Windows, or custom configurations, see the [full Setup Guide](/v2/orchestrators/setup/guide).
</Note>

<CustomDivider />

## Prerequisites

Before you start, confirm:

- **NVIDIA GPU** visible via `nvidia-smi` (driver 525+)
- **Docker Engine** installed ([docs.docker.com](https://docs.docker.com/engine/install/))
- **NVIDIA Container Toolkit** installed — verify: `docker run --rm --gpus all nvidia/cuda:12.0.0-base-ubuntu22.04 nvidia-smi`
- **Port 8935 TCP** open to the public internet
- **Arbitrum RPC URL** from [Alchemy](https://www.alchemy.com) or [Infura](https://infura.io) (free tier is fine)
- **ETH on Arbitrum** for gas (a few dollars)
- **LPT on Arbitrum** for staking (check [Explorer](https://explorer.livepeer.org) for the current active set threshold)

See [Setup Checklist](/v2/orchestrators/setup/rcs-requirements) for detailed requirements. See [Arbitrum Exchanges](/v2/orchestrators/resources/arbitrum-exchanges) for where to get ETH and LPT on Arbitrum.

<CustomDivider />

## Step 1 — Pull the Docker image

```bash icon="terminal"
docker pull livepeer/go-livepeer:0.8.9
```

Verify:

```bash icon="terminal"
docker run --rm livepeer/go-livepeer:0.8.9 livepeer -version
```

<CustomDivider />

## Step 2 — Create your data directory

```bash icon="terminal"
mkdir -p ~/.lpData
```

This is where go-livepeer will store your keystore and node data.

<CustomDivider />

## Step 3 — Start the node

Replace the three placeholders below with your values:

<StyledTable variant="bordered">
  <TableRow header>
    <TableCell header>Placeholder</TableCell>
    <TableCell header>Value</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>`<YOUR_RPC_URL>`</TableCell>
    <TableCell>Your Alchemy or Infura Arbitrum endpoint</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>`<YOUR_PUBLIC_IP>`</TableCell>
    <TableCell>Your server's public IP address</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>`<PRICE>`</TableCell>
    <TableCell>Price per pixel in wei (e.g. `1200` — check [livepeer.tools](https://livepeer.tools) for current market rates)</TableCell>
  </TableRow>
</StyledTable>

```bash icon="terminal"
docker run -d \
  --name livepeer \
  --gpus all \
  --network host \
  -v ~/.lpData:/root/.lpData \
  livepeer/go-livepeer:0.8.9 \
  -network arbitrum-one-mainnet \
  -ethUrl <YOUR_RPC_URL> \
  -orchestrator \
  -transcoder \
  -nvidia 0 \
  -serviceAddr <YOUR_PUBLIC_IP>:8935 \
  -pricePerUnit <PRICE> \
  -pixelsPerUnit 1000000000000 \
  -monitor
```

Check the logs:

```bash icon="terminal"
docker logs -f livepeer
```

You should see:

``` icon="terminal"
Transcoding on Nvidia GPU 0
```

and:

``` icon="terminal"
Listening for RPC on :8935
```

<Warning>
On first run, go-livepeer will create a new Ethereum account and prompt you for a passphrase. **Remember this passphrase** — you need it to restart the node. The keystore is saved in `~/.lpData/arbitrum-one-mainnet/keystore`.
</Warning>

If you already have an Ethereum account, add `-ethKeystorePath` and `-ethPassword` flags. See [Configure](/v2/orchestrators/setup/r-configure) for details.

<CustomDivider />

## Step 4 — Verify the connection

Confirm you are connected to Arbitrum:

```bash icon="terminal"
curl -s -X POST -H "Content-Type: application/json" \
  --data '{"jsonrpc":"2.0","method":"eth_chainId","params":[],"id":1}' \
  <YOUR_RPC_URL>
```

Expected: `"result":"0xa4b1"` (Arbitrum One, chain ID 42161).

Confirm your node is publicly reachable (run from a **different machine**):

```bash icon="terminal"
curl -k https://<YOUR_PUBLIC_IP>:8935/status
```

If this times out, check your firewall and port forwarding. See [Verify](/v2/orchestrators/setup/test) for troubleshooting.

<CustomDivider />

## Step 5 — Activate on the network

Open `livepeer_cli` in a second terminal:

```bash icon="terminal"
docker exec -it livepeer livepeer_cli
```

Select **"Invoke multi-step 'become an orchestrator'"** and follow the prompts:

<StyledTable variant="bordered">
  <TableRow header>
    <TableCell header>Prompt</TableCell>
    <TableCell header>Recommended value</TableCell>
    <TableCell header>Notes</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>Reward cut</TableCell>
    <TableCell>`10`</TableCell>
    <TableCell>You keep 10% of LPT rewards, delegators share 90%</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>Fee cut</TableCell>
    <TableCell>`95`</TableCell>
    <TableCell>You keep 95% of ETH fees</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>Pixels per unit</TableCell>
    <TableCell>`1`</TableCell>
    <TableCell>Accept the default</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>Price per unit</TableCell>
    <TableCell>Same as your `-pricePerUnit`</TableCell>
    <TableCell>Must be consistent with your startup flag</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>Service address</TableCell>
    <TableCell>`<YOUR_PUBLIC_IP>:8935`</TableCell>
    <TableCell>Must match your `-serviceAddr`</TableCell>
  </TableRow>
  <TableRow>
    <TableCell>Stake amount</TableCell>
    <TableCell>Your LPT amount in LPTU</TableCell>
    <TableCell>1 LPT = `1000000000000000000` LPTU. To stake 100 LPT, enter `100000000000000000000`</TableCell>
  </TableRow>
</StyledTable>

<Note>
Two transactions will be submitted: `approve` (authorises the staking contract) and `bond` (stakes your LPT). Wait for both to confirm on Arbitrum.
</Note>

<CustomDivider />

## Step 6 — Verify activation

Your node joins the active set at the start of the **next round** (~24 hours after activation).

1. Go to [Livepeer Explorer](https://explorer.livepeer.org)
2. Search for your Ethereum address
3. Confirm status shows **Active**

Check your logs for job activity:

```bash icon="terminal"
docker logs livepeer 2>&1 | grep -i "got segment"
```

<Accordion  title="Not receiving jobs?" icon="triangle-exclamation">

**Service address not reachable** — the #1 cause. Re-test port 8935 from an external machine.

**Price too high** — gateways filter by max price. Check current rates on [livepeer.tools](https://livepeer.tools).

**Not in the active set** — you need top-100 stake. Check your rank on Explorer.

**Just activated** — wait for the next round (~24 hours).

For more, see [Verify Your Setup](/v2/orchestrators/setup/test) and the [FAQ](/v2/orchestrators/resources/faq).

</Accordion>
