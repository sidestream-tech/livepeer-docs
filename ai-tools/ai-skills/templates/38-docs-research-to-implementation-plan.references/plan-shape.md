# Plan Shape

The implementation artifact should be decision-complete and short enough to hand to another agent.

## Required Sections

1. Summary
2. Current Findings
3. Work Phases
4. Validation
5. Risks And Follow-Up

## Phase Contract

Each phase should state:

- target outcome
- claim families affected
- files or systems likely to change
- why the phase belongs before or after other phases
- exact validation gates

## Separation Rules

- Content changes:
  - page wording, removals, clarifications, propagation edits
- Registry changes:
  - claim-family coverage, canonical owners, truth modes, discovery config
- Evidence-adapter changes:
  - source discovery, ranking, fetch behavior, reporting annotations

Do not blur these together in one generic task list.
